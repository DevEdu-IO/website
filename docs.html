How containers and the DevEdu Code assistant work — and the open-source images you can run and extend yourself.

What is a container?

A container is your own private development environment that runs in the cloud and opens right in your web browser — like a fresh computer set up just for your course. It already has the editor (VS Code), language tools, and a terminal you need, so there's nothing to download or configure. Each student gets their own; your work is private and saved between sessions.

Launching one

  1. Open your Course, then an Environment inside it.
  2. Click New container and confirm — we give it a fun, unique name automatically, and it starts up on its own.
  3. Open the Editor link to write and run code.

Opening your container

Just click the Editor or App button. As long as you're signed in to DevEdu, your container opens straight away. Teachers can open any of their students' containers the same way.

Use Google Chrome. The browser editor works best in Chrome; other browsers may not behave correctly.

Editor & app URLs

Every container gives you two web addresses:

  • Editor — editor-<name>-<env>.lab.devedu.io, where you write and run code (the <name> is the fun name we assign).
  • App — for environments that run a web app, app-<name>-<env>.lab.devedu.io shows your running program (it listens on port 3000). You'll usually just click the Editor and App buttons rather than type these.

Credits & time

DevEdu is free for teachers. Running a container uses container-days from a student's balance: a credit is $99 for 140 days of container time. Idle containers sleep automatically, and deleting a container before it expires returns the unused days to the student's balance.

DevEdu Code AI assistant

DevEdu Code is an AI pair-programmer built into every container (and available as a command-line tool). Ask it to explain an error, walk through a concept, write a function, or refactor your code. It can read the files in your workspace, make edits, and run commands — but it always asks for your approval before changing anything or running something.

In your container

  1. Click the </> DevEdu Code icon in the left activity bar.
  2. Type a question or request — e.g. "why am I getting a segfault?"
  3. When it wants to write a file or run a command, you get an Apply / Run button — nothing happens until you approve it.

On your own machine

The same assistant is a command-line tool. On macOS or Linux, install it with one line:

curl -fsSL https://raw.githubusercontent.com/DevEdu-IO/code/main/install.sh | sh

On Windows, download devedu-windows-amd64.exe from the releases page. Then run devedu for an interactive session, or devedu "your question" for a one-off. It's open source: github.com/DevEdu-IO/code.

Auth & cost

In a DevEdu container the assistant is already signed in to your account. On your own machine it asks for your API key once (Account → API key) and stores it securely. Usage draws from your account's token balance; every student gets a chunk free on enrolling in a course, with top-ups available.

Run one yourself

You only need Docker. Pull an image and start it — then open the editor in your browser:

docker pull deveduio/cpp:latest
docker run -d --name devedu-cpp -p 8080:80 deveduio/cpp:latest
# now open http://localhost:8080

The Django image also serves your app on port 3000:

docker run -d -p 8080:80 -p 3000:3000 deveduio/django:latest

Images are multi-arch, so they run on Intel/AMD (amd64) and Apple Silicon / ARM (arm64).

Build your own

Need a different stack? Clone a repo, edit the Dockerfile, and build — or start from scratch with code-server and add the DevEdu Code .vsix. Teachers can host custom images on their own infrastructure or on DevEdu's.

git clone https://github.com/DevEdu-IO/docker-django.git
cd docker-django
# edit Dockerfile.amd64 / Dockerfile.arm64, then:
./build.sh --load     # build for your machine

Questions or want a custom environment built with you? help@devedu.io.
